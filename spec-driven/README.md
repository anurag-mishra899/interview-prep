## 🔍 Key Concepts Reference

### The Three Maturity Levels

| Level | Description | When to Use | Commitment |
|-------|-------------|-------------|------------|
| **Spec-First** | Write specs, implement, then delete specs | Prototypes, experiments, learning | Low |
| **Spec-Anchored** | Maintain specs alongside code | Production features, documentation needed | Medium |
| **Spec-as-Source** | Only edit specs, code is generated | Strategic components, frequent tech pivots | High |

**See:** [ENTERPRISE_ADOPTION_GUIDE.md - Section 5](ENTERPRISE_ADOPTION_GUIDE.md#5-transition-across-maturity-levels)

---

### The Four Pillars of SDD

1. **Specifications Are Executable** - Not documentation, but contracts
2. **Natural Language > Formal Syntax** - Human-readable, AI-understandable
3. **Specifications Evolve with Code** - Versioned, reviewed, maintained
4. **Progressive Formalization** - Match rigor to feature importance

**See:** [VENDOR_AGNOSTIC_FRAMEWORK.md - Part 1.1](VENDOR_AGNOSTIC_FRAMEWORK.md#11-the-four-pillars-of-sdd)

---

### Constitutional Principles

**Universal principles for quality:**

1. **Specification Primacy** - Specs are source of truth
2. **Test-First Development** - Tests validate specs
3. **Contract-First Design** - APIs/data defined before implementation
4. **Simplicity First** - Justify complexity
5. **Security First** - Threat modeling mandatory
6. **Progressive Formalization** - Match formality to importance

**See:** [VENDOR_AGNOSTIC_FRAMEWORK.md - Part 6.2](VENDOR_AGNOSTIC_FRAMEWORK.md#62-constitutional-principles-greenfield--brownfield)

## ⚠️ Critical Insights

### From Martin Fowler's Analysis:

1. **"Reviewing markdown over reviewing code"**
   - Spec review burden may equal or exceed code review
   - Need efficient spec review practices

2. **Non-determinism persists**
   - Despite templates, AI doesn't follow all instructions
   - Strong testing essential

3. **Parallels to Model-Driven Development (MDD)**
   - Risk of "inflexibility AND non-determinism"
   - Be cautious with spec-as-source

4. **Brownfield challenges**
   - Harder to retrofit than greenfield
   - Incremental adoption critical

**Full analysis:** [SPEC_DRIVEN_DEVELOPMENT_RESEARCH.md - Section 8](SPEC_DRIVEN_DEVELOPMENT_RESEARCH.md#8-critical-observations--open-questions)
